# Protocol Assembly Skill

## Procedure

1. Collect final revised disagreement items.
2. Normalize numbering and clause references.
3. Preserve current wording, proposed wording and rationale.
4. Add unresolved questions and lawyer-review checklist.
5. Produce JSON first, Markdown render second.

## Markdown Table Columns

- Clause;
- Current wording;
- Proposed wording;
- Rationale;
- Priority;
- Fallback;
- Review flag.

## Guardrails

Do not mark a protocol as approved. Approval belongs to the human user.

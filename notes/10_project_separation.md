# Статус самостоятельного проекта

Дата: 14.05.2026

Обновлено: 16.05.2026

## Решение

`Jurist` ведется как самостоятельный проект для проверки договоров, чтения правовых источников и подготовки протоколов разногласий.

## Границы

Jurist отвечает за:

- проверку договоров;
- подготовку протоколов разногласий;
- поиск открытых правовых источников;
- чтение судебной практики;
- работу с DaMIA по арбитражным делам;
- работу с OpenAI и OpenRouter моделями через локальный runtime;
- подготовку юридических файлов проверки;
- будущую интеграцию с Google Drive.

## Структура

- `src/contract_protocols`;
- `tests`;
- `config`;
- `roles`;
- `skills`;
- `schemas`;
- `notes`;
- `examples`;
- пустые директории `storage/cases` и `storage/input` с `.gitkeep`.

## Правила чистоты проекта

- реальные проверки не коммитятся;
- входные пользовательские договоры не коммитятся;
- локальные секреты и `.env` не коммитятся;
- служебные директории `__pycache__` не коммитятся;
- внешние записи в Google Drive выполняются только после явного подтверждения пользователя.

## Следующие шаги

- хранить ключи DaMIA, OpenAI и OpenRouter только в локальном `.env`;
- использовать DaMIA в workflow только по прямому указанию `--enable-damia`;
- перед первым live-запуском дня выполнять `jurist models health-check`;
- выполнить первый небольшой `run-live` на примере из `examples/`;
- добавить автоматический post-run quality report;
- поддерживать заметки Jurist без ссылок на посторонние продуктовые домены.
